# 3000词 · 高考拔高（手机端 Web 学习应用）

把《【高考拔高】3000词》的学习法做成一个**手机优先、离线可用、Supabase 云同步**的网页应用。
当前已接入 **Day 1–30 全量 3000 词**，四个学习模式 + 遗忘曲线复习调度 + 错题本 + 打卡 + 发音全部可用。

## 这份材料的规则（应用严格遵循）

- 全书 **30 天 × 100 词 = 3000 词**，每词带难度星级：无星=**基础**(小学/初中)、`*`=**必修**(高中必修)、`**`=**选修**(高中选修)。
- 每词配一句**强语境英文句**（目标词用括号标出）+ **中文翻译**。
- 学习四步：**① 自查**（一眼判断会不会）→ **② 强语境**（不看中文猜词义）→ **③ 对照中文自讲**（校准+讲解）→ **④ 重复**；**进阶：中译英**（看中文写英文，把"消极识别"变"积极调用"）。

## 应用如何把"学习法"变成"学习-考核-反馈"闭环

| 底栏 | 对应书中 | 做什么 |
|---|---|---|
| 📖 学习 | 第 1–3 步 | **按组学习**（每天 100 词分 5 组 × 20 词）。每组：**① 自查**——逐个看词，「毫不犹豫一眼认出」的选「秒认」→ **弹出正确意思核对**，确认认对才跳过并排入复习；核对发现认错、或本就犹豫的，进入 → **② 强语境**：挖空句→猜→显词朗读→对照中文→自评「猜对/没猜对」 |
| ✅ 考核 | 识别检验 | 看词+句选正确中文（4 选 1），自动判分 |
| ✍️ 进阶 | 进阶·中译英 | 看中文写英文 → 对照原句 → 按三档自评：完全一样 / 含目标词且语义语法正确 / 有错误 |
| 📊 评估 | 学习报告 | 汇总学习、考核、进阶记录，生成近 7 天表现、分组诊断、薄弱词和下一步建议 |
| 🔁 错题本 | 反馈 | 自动收集猜错/答错的词，可一键专项攻克；答对一次即移出 |
| 🏠 首页 | 反馈 | 今日新词 / 待复习 / 掌握率 / 连续打卡 streak / 100 词掌握热力图 |

**遗忘曲线（间隔重复）引擎**：Leitner 盒 + SM-2 简化，间隔约 `10分钟 → 1 → 2 → 4 → 7 → 15 → 30 天`。
- 每词记录：盒号、下次到期时间、复习次数、遗忘次数、错题标记、掌握状态。
- 首页顶部推「到期复习」；下面 5 个分组卡各自显示进度（未开始/学习中/复习中/已掌握），按组推进。
- 当前已会且不是错题的词计入页面**掌握/已会**进度；盒号继续用于安排后续间隔复习。秒认核对通过的词进较高复习盒，隔几天再确认一次。

**多用户 + 学习码云同步（离线优先）**：
- 进入应用先选/建用户；每个学生自动获得一个**学习码**（如 `XD-4821`，右上角 `⋯` 里查看/复制）。
- **离线优先**：学生随时离线都能学，进度先存本机；一联网就自动同步到云端（首页有同步状态提示）。
- **换设备**：在登录页「用学习码登录」输入码，即可把云端进度拉到新设备继续。
- **家长查看**：登录页「家长 · 查看孩子进度」输入孩子的学习码，看到**只读**看板（掌握率、分组进度、薄弱词、打卡）。

**云后端可插拔**（`app.js` 顶部 `CLOUD`）：
- `backend: 'supabase'`（当前默认）——已接入真云端，跨设备同步、家长在自己手机看进度。当前 Supabase 项目已验证可访问 `learners` 表。
- `backend: 'mock'`——用浏览器本地模拟云，无需注册即可完整演示学习码/家长看板/换设备；但**不跨真实设备**（仅同一浏览器内演示）。
- 如需重新接入另一个 Supabase 项目：
  1. 在 [supabase.com](https://supabase.com) 建一个 project。
  2. SQL Editor 里运行本仓库的 [`supabase.sql`](supabase.sql) 建表。
  3. 确认 `index.html` 已加载 `@supabase/supabase-js`。
  4. `app.js` 顶部把 `CLOUD.backend` 设为 `'supabase'`，填入 project URL 和公开 anon key。
  5. 部署后即生效，学生端、家长端用同一学习码就能跨设备同步。
> 安全模型：家庭级「凭学习码即可读写」。学习进度无隐私敏感信息，适合家庭/小范围使用；需要更强隔离见 `supabase.sql` 末尾的 RPC 方案。

## 使用方式

### 方式 A：手机上直接用（推荐）
把整个文件夹放到任意静态托管（GitHub Pages / Netlify / Vercel / 自建），手机浏览器打开 `index.html`：
- Safari/Chrome 菜单 →「添加到主屏幕」，即可像 App 一样全屏、离线使用。
- 进度先存在手机本地（localStorage），联网后自动同步到 Supabase 云端。

### 方式 B：本地预览
```bash
cd English-3000-words
python3 -m http.server 4173
# 浏览器打开 http://localhost:4173
```
也可直接双击 `index.html` 用 `file://` 打开（发音、学习都可用；仅「添加到主屏幕/Service Worker 离线」需要 http 托管）。

## 全量词表说明

- `data/day1.js` 到 `data/day30.js` 对应考纲第 1–3000 词。
- Day 2–30 由本项目 `scripts/extract_pdf_days.py` 从原书 PDF 自动抽取生成，并校验为 **30 天 × 100 词 = 3000 词**。
- `level` 星级、英文强语境句、中文译文来自原书；`gloss` 是为「英译中」选择题补充的中文简义，Day 2–30 由脚本从译文中程序化提取/压缩。
- 右上角 `⋯` 菜单可切换 Day 1–30；同一学生账号下，各 Day 的学习进度会按词条 ID 分开记录。

## 缓存说明
`index.html` 里 `styles.css?v=15 / app.js?v=18 / data/dayN.js?v=11` 的 `?v=` 是版本号。每次改动这些文件后把 `v` 数字 +1，用户浏览器即会拉取新版本，避免旧缓存；同时更新 `sw.js` 里的 `CACHE` 名称，让已添加到主屏幕的用户刷新离线缓存。

## 开发校验清单

- 词表更新后运行 `node scripts/validate_data.js`，确认 30 天、3000 词、每天 100 词、例句均含目标词括号。
- 从 PDF 重新抽取 Day 2–30 时运行 `python3 scripts/extract_pdf_days.py`，不要手工批量复制。
- 修改 `app.js` 后运行 `node --check app.js`。
- 修改静态资源后同步提升 `index.html` 的 `?v=` 版本和 `sw.js` 的 `CACHE` 名称。
- Service Worker 只缓存同源静态资源，不缓存 Supabase、CDN 或其他 API 响应。
- 公开部署后优先使用 Supabase RPC：前端调用 `learner_get` / `learner_upsert`，表级 anon 权限应收紧。
